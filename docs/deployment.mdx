A **deployment** is a host-side backend that provisions sandboxes. The
user picks one with `agentix deploy <name> --image …`; in code, you
instantiate the class and pass it to `session(...)`. Built-ins:
`local` (Docker), `daytona`, `e2b`. `pip install agentix-deployment-*`
adds another.

## Using one

<Tabs>
  <Tab title="CLI">
    ```bash
    agentix deploy local --image my-agent:0.1.0
    agentix deploy fly   --image my-agent:0.1.0    # after `pip install agentix-deployment-fly`
    ```
  </Tab>
  <Tab title="Code">
    ```python
    from agentix import RuntimeClient, SandboxConfig
    from agentix.deployment.base import session
    from agentix.deployment.docker import DockerDeployment

    deployment = DockerDeployment()
    config = SandboxConfig(image="my-agent:0.1.0")
    async with session(deployment, config) as sandbox:
        async with RuntimeClient(sandbox.runtime_url) as c:
            ...
    ```
  </Tab>
</Tabs>

`session(deployment, config)` creates the sandbox on entry and deletes
it on exit.

## Writing one

A deployment backend is a class that implements three methods. No
inheritance — the framework uses structural typing.

<Tabs>
  <Tab title="Plugin source">
    ```python title="my_deploy/__init__.py"
    import os
    from agentix.deployment.base import Sandbox
    from agentix.idents import SandboxId
    from agentix.models import SandboxConfig, SandboxInfo

    class FlyDeployment:
        def __init__(self) -> None:
            # Backends take NO constructor args — read config from env.
            self._token = os.environ["FLY_API_TOKEN"]

        async def create(self, config: SandboxConfig) -> Sandbox: ...
        async def delete(self, sandbox_id: SandboxId) -> None: ...
        async def get(self, sandbox_id: SandboxId) -> SandboxInfo: ...
    ```
  </Tab>
  <Tab title="pyproject.toml">
    ```toml
    [project]
    name = "agentix-deployment-fly"
    version = "0.1.0"

    [project.entry-points."agentix.deployment"]
    fly = "my_deploy:FlyDeployment"
    ```
  </Tab>
</Tabs>

After `pip install`, `agentix deploy fly --image my-agent:0.1.0` works
with no framework changes.

## Configuration

Backend config reads from environment variables:

| Variable                          | Used by              | Purpose                                  |
| --------------------------------- | -------------------- | ---------------------------------------- |
| `DAYTONA_API_KEY`                 | `daytona`            | API auth                                 |
| `E2B_API_KEY` / `E2B_TEMPLATE_ID` | `e2b`                | API auth + template                      |
| `AGENTIX_BIND_PORT`               | runtime server       | Bind port (default 8000)                 |

Read env vars in `__init__` and fail fast if any are missing — the
error surfaces at CLI time instead of inside the first `create()`.
