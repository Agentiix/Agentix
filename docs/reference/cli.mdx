## `agentix build`

Build one namespace image, or bundle several into a single image.

```bash
# One namespace — tag auto-derived from pyproject.toml
agentix build ../Agentix-Runtime-Basic
agentix build ../Agentix-Runtime-Basic --dry-run
agentix build ../Agentix-Runtime-Basic -o my-bundle:dev

# Many namespaces — explicit -o required
agentix build ../Agentix-Runtime-Basic ../claude-code -o my-agent:0.1.0
agentix build ../Agentix-Runtime-Basic ./my-experimental-namespace -o demo:dev
agentix build ../Agentix-Runtime-Basic ../claude-code --dry-run -o sandbox:dev
```

Each spec accepts:

- **Path:** `../Agentix-Runtime-Basic` or any directory with a `pyproject.toml`.
- **PyPI dist:** `agentix-runtime-basic` (currently raises `NotImplementedError`).

`-o / --output / --tag` is the output image tag. With one spec it's
optional (auto-derived as `agentix/<short>:<version>`). With multiple
specs it's required. `--dry-run` stages the docker build context to
`./build/<name>/` and prints the path instead of invoking docker.

## `agentix deploy`

Provision a sandbox.

```bash
agentix deploy <backend> --image <tag>
agentix deploy local --image my-agent:0.1.0
agentix deploy local --image my-agent:0.1.0 --detach
agentix deploy daytona --image docker.io/me/my-agent:0.1.0    # currently NotImplementedError
```

`<backend>` is any registered `agentix.deployment` plugin. Built-ins:
`local`, `daytona`, `e2b`.

Foreground by default: prints `runtime_url`, waits for Ctrl-C, tears
the sandbox down. `--detach` exits after `create()` and prints the
handle.

## `agentix check`

List installed namespaces and smoke-import each. Non-zero exit code
if any fail.

```bash
agentix check
```

## Configuration

Most config reads from environment variables.

| Variable                          | Used by                    | Purpose                                |
| --------------------------------- | -------------------------- | -------------------------------------- |
| `AGENTIX_BIND_PORT`               | runtime server             | Bind port (default 8000)               |
| `AGENTIX_UPLOAD_ROOT`             | `agentix.files` namespace  | Sandbox-side root for file I/O         |
| `DAYTONA_API_KEY`                 | `daytona` deployment       | API auth                               |
| `E2B_API_KEY` / `E2B_TEMPLATE_ID` | `e2b` deployment           | API auth + template                    |

Third parties that want their own top-level verb should ship a
separate `console_scripts` binary (`agentix-yourcmd`) rather than
registering against the central dispatcher.
