---
title: Architecture
description: How bundle dependencies and remote calls fit together.
---

Agentix has two core pieces:

1. **Bundle** builds one runtime image containing user code,
   integrations, Python dependencies, and optional system binaries.
2. **Remote calls** run installed functions inside that image with
   `RuntimeClient.remote(fn, ...)`.

Bundle decides what code exists in the sandbox. `client.remote(fn, ...)`
chooses which installed function to call.

## User Model

Pass a normal imported function:

```python
from agentix import RuntimeClient
from app import run

async with RuntimeClient(sandbox.runtime_url) as client:
    result = await client.remote(run, input="hello")
```

Importing the module first also works:

```python
import app

result = await client.remote(app.run, input="hello")
```

## Bundle

`agentix build [path]` installs the project into the runtime venv:

```bash
/nix/runtime/bin/pip install --no-cache-dir /src/project
```

That install brings in the user project, direct dependencies,
transitive dependencies, and integration modules declared in
`pyproject.toml`.

Runtime layout:

```text
/nix/runtime/
├── bin/
│   ├── python
│   ├── pip
│   └── agentix-server
└── lib/python3.11/site-packages/
    ├── agentix/
    ├── agentix/bash/
    ├── agentix/swebench/
    └── app.py
```

If the project includes `default.nix`, `agentix build` adds a Nix
builder stage and symlinks the derivation's `bin/*` into
`/nix/runtime/bin/`. Worker subprocesses use:

```text
/nix/runtime/bin:/usr/local/bin:/usr/bin:/bin
```

## Remote Calls

The client turns the function object into a target string:

```python
target = f"{fn.__module__}::{fn.__name__}"
```

Example:

```python
from agentix.swebench import run

score = await client.remote(run, instance=inst, patch=patch)
```

Wire payload:

```python
{
    "target": "agentix.swebench::run",
    "args": [],
    "kwargs": {"instance": inst, "patch": patch},
}
```

The runtime worker imports `agentix.swebench` inside the sandbox and
calls `run(...)`.

Args, kwargs, return values, and stream items travel as msgpack payloads.
The client uses the local function signature to encode inputs; the
worker resolves the imported function signature and validates/coerces
inputs before calling it.

## Flow

```text
Host
  RuntimeClient.remote(fn, ...)
    read fn.__module__ and fn.__name__
    build "module.path::function_name"
    detect unary / stream / bidi
    encode args and kwargs
      |
      v
Sandbox
  agentix-server
      |
      v
Single runtime worker process
  import module.path
  call function_name(*args, **kwargs)
```

## Call Shapes

| Function shape | Transport | Client usage |
| --- | --- | --- |
| normal async function | HTTP `POST /_remote` | `await client.remote(fn, ...)` |
| async generator | Socket.IO stream | `async for item in client.remote(fn, ...)` |
| async generator with `Channel[T]` input | Socket.IO bidi | send through `Channel`, receive with `async for` |

## Mental Model

```text
Bundle = what code and dependencies exist in the sandbox
client.remote(fn) = which installed function to call
Worker = where the function executes
```
